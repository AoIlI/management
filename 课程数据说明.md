## 📝 使用说明

### 1. 执行SQL脚本

```sql
-- 在MySQL中执行课程数据插入脚本.sql文件
source 课程数据插入脚本.sql;
-- 或者直接复制SQL语句到数据库管理工具执行
```

### 2. 注意事项

1. **日期调整**：脚本中的日期为2024-12-16开始，请根据实际需要调整
2. **教练ID**：确保数据库中存在对应的教练记录（S001-S005）
3. **ID冲突**：如果表中已有数据，请检查class_id是否冲突
4. **数据验证**：执行后建议查询验证数据是否正确插入

### 3. 查询验证

```sql
-- 查看所有课程
SELECT * FROM fitness_classes ORDER BY schedule_time;

-- 查看课程统计
SELECT class_name, COUNT(*) as count, SUM(max_capacity) as total_capacity, SUM(current_enrollment) as total_enrollment
FROM fitness_classes
WHERE coursestatus = 'ACTIVE'
GROUP BY class_name;

-- 查看每个教练的课程数
SELECT coach_id, COUNT(*) as course_count
FROM fitness_classes
WHERE coursestatus = 'ACTIVE'
GROUP BY coach_id;

-- 查看每天的课程数
SELECT DATE(schedule_time) as date, COUNT(*) as course_count
FROM fitness_classes
WHERE coursestatus = 'ACTIVE'
GROUP BY DATE(schedule_time)
ORDER BY date;
```

### 4. 数据特点

- **课程容量**：15-30人不等，根据课程类型设置
- **当前报名人数**：设置为0到接近最大容量的合理数值，方便测试预约功能
- **课程状态**：全部设置为'ACTIVE'，表示可预约
- **课程时长**：45分钟、60分钟、75分钟，符合实际健身课程时长
